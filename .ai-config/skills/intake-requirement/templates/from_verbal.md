### 推荐的输出物规范

根据最终等级，以下内容为必需：

**S等级（轻量版）**：本需求文档（基础6字段）即可，无需补充

**M等级（标准版）**：本需求文档需补充足以达到10个必填字段

**L等级（完整版）**：本需求文档需补充以下内容：
- [ ] 业务背景（background）：背景说明
- [ ] 目标（goal）：1-3条可量化目标
- [ ] 干系人列表（stakeholders）：提需人、技术负责人、影响方等
- [ ] 非功能需求（non_functional）：性能/安全/兼容性等具体要求
- [ ] 已知风险点（risks）：技术或业务风险 + 应对方案
- [ ] 技术思路骨架（tech_sketch）：受影响模块、初步架构思路、关键决策点
- [ ] UI原型（如有前端变更）：原型图链接或内嵌

**XL等级（超大型）**：本需求文档需补充：
- [ ] 迭代拆分计划（iteration_plan）：将XL拆分为若干L/M子需求
- [ ] 里程碑计划（milestones）：各阶段目标 + 时间节点
- [ ] 所有L等级的补充内容
- [ ] UI原型（必须，涉及前端时）

> 下一步：根据最终等级，补充相应的缺失字段，准备进入【Step 4】质量检查。
