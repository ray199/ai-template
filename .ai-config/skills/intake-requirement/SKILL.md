# 技能：需求接入与结构化

## 技能描述

将任意形式的原始需求（口头描述、飞书截图、简短消息、会议纪要等）转化为标准结构化需求文档，并自动检测信息缺失项，输出澄清问题清单。

## 触发指令

- `/intake` - 接入并结构化一条新需求
- `/intake-check` - 检查已有需求文档的完整性
- `/intake-list` - 列出当前所有需求及其状态

## 处理流程

```
原始输入
   ↓
[Step 1] 识别需求来源类型（研发自驱 / 业务方）
   ↓
[Step 2] 选择对应模板，提取关键信息填充字段
   ↓
[Step 3] 完整性检查：逐一核查必填字段
   ↓
[Step 4] 输出结构化文档 + 缺失字段澄清清单
   ↓
[Step 5] 等待确认后，分配 need_id，写入 docs/requirements/backlog/
```

## 输入格式

直接粘贴原始内容即可，支持：
- 飞书消息文本
- 口头描述的文字记录
- 会议纪要片段
- 内部技术提案

```
/intake

[原始需求内容，直接粘贴]
```

## 输出格式

### 结构化文档（成功时）

输出填写完整的需求文档（使用对应模板），并在末尾附上：

```markdown
---
## ⚠️ 待澄清项

以下字段信息不足，请补充后方可进入技术设计阶段：

1. **[字段名]**：[当前内容或"未提及"] → 需要确认：[具体问题]
2. **[字段名]**：[当前内容] → 存在歧义：[具体问题]

确认完成后，请回复 `/intake-confirm REQ-XXXXXXXX` 以推进流程。
```

### 无法解析时

```markdown
## ❌ 无法完成结构化

原始内容信息量不足，无法识别以下核心要素：
- [ ] 需求目标不明确
- [ ] 功能边界未定义
- [ ] 其他：[说明]

建议：请参考模板 `.ai-config/skills/intake-requirement/templates/from_verbal.md` 补充信息。
```

## 完整性检查规则

| 字段 | 检查规则 |
|---|---|
| `title` | 不超过20字，包含动词+对象 |
| `background` | 至少说明业务场景和当前问题 |
| `goal` | 必须可量化（有数字或可验证的状态） |
| `scope` | 必须同时包含"包含"和"不包含"两部分 |
| `priority` | 只接受 P0 / P1 / P2，需说明定级理由 |
| `deadline` | 具体日期或明确的里程碑，不接受"尽快" |
| `acceptance` | 至少2条，格式为"当...时，系统应..." |
| `feasibility` | 必须有初步判断，研发自驱需求须自评 |
