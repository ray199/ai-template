# 代码审查清单

## 代码质量
- [ ] 代码可读性：变量命名清晰，函数职责单一
- [ ] 代码复杂度：避免过度嵌套，函数长度适中
- [ ] 代码重复：无重复代码，已提取公共逻辑
- [ ] 代码注释：关键逻辑有适当注释
- [ ] 错误处理：异常情况有合理处理

## 安全性
- [ ] SQL注入：使用参数化查询或ORM
- [ ] XSS防护：对用户输入进行转义
- [ ] CSRF防护：实现CSRF令牌验证
- [ ] 密码存储：使用安全的哈希算法
- [ ] 权限控制：实现基于角色的访问控制
- [ ] 敏感信息：无硬编码的API密钥或密码

## 性能
- [ ] 内存使用：避免内存泄漏
- [ ] 网络请求：减少不必要的请求，实现缓存
- [ ] 渲染性能：优化组件渲染，避免不必要的重渲染
- [ ] 算法效率：使用合适的算法和数据结构
- [ ] 资源加载：优化资源加载顺序和大小

## 最佳实践
- [ ] 遵循编码规范：符合项目的代码风格
- [ ] 测试覆盖：有适当的单元测试和集成测试
- [ ] 模块化：代码结构清晰，模块划分合理
- [ ] 依赖管理：依赖版本合理，无安全漏洞
- [ ] 文档：代码和API有适当的文档

## 前端特有检查
- [ ] 响应式设计：适配不同屏幕尺寸
- [ ] 无障碍性：符合WCAG标准
- [ ] 浏览器兼容性：支持主流浏览器
- [ ] 状态管理：状态管理合理，避免过度使用全局状态
- [ ] 路由管理：路由配置清晰，导航体验良好

## 后端特有检查
- [ ] API设计：RESTful API设计规范
- [ ] 数据库设计：表结构合理，索引优化
- [ ] 并发处理：处理并发请求，避免竞态条件
- [ ] 日志记录：关键操作有日志记录
- [ ] 配置管理：环境配置合理，敏感配置分离
