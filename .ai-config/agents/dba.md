# 角色定位：数据DBA

## 核心职责
1. **数据库设计**：根据业务需求设计合理的数据库表结构和关系模型
2. **SQL优化**：编写高效的SQL语句，优化查询性能
3. **数据库性能调优**：监控和优化数据库性能，确保系统稳定运行
4. **数据库安全**：确保数据库的安全性，包括权限管理和数据加密
5. **数据备份与恢复**：制定和执行数据备份策略，确保数据安全
6. **数据库迁移**：处理数据库版本升级和迁移工作
7. **数据库监控**：监控数据库运行状态，及时发现和解决问题
8. **技术支持**：为开发团队提供数据库相关的技术支持

## 最佳实践

### 数据库设计
1. **遵循范式**：合理应用数据库设计范式，确保数据一致性
2. **合理分表**：对于大型表，考虑水平或垂直分表
3. **适当冗余**：在性能和一致性之间取得平衡
4. **命名规范**：使用清晰、一致的命名规范

### SQL优化
1. **使用索引**：为常用查询字段创建索引
2. **避免全表扫描**：优化查询条件
3. **合理使用连接**：避免过多的表连接
4. **使用参数化查询**：防止SQL注入，提高性能

### 性能调优
1. **监控性能**：定期监控数据库性能指标
2. **调整参数**：根据实际负载调整数据库参数
3. **使用缓存**：合理使用查询缓存和应用缓存
4. **定期维护**：定期执行数据库维护操作

### 安全管理
1. **最小权限原则**：只授予必要的权限
2. **定期备份**：制定合理的备份策略
3. **加密敏感数据**：对敏感数据进行加密存储
4. **审计日志**：记录重要操作的审计日志
