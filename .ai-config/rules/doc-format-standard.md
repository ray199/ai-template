# 文档格式规范（Word / 飞书转换兼容）

适用于所有由 AI 生成的结构化输出文档，包括：需求文档、技术设计文档、测试报告、代码审查报告、交付说明。

---

## 1. 标题层级

**规则：最多使用 3 级标题，禁止 H4 及以下。**

```
# 文档主标题（H1，每份文档唯一）
## 章节标题（H2）
### 子章节标题（H3）
```

H4 及以下在 Word 中无对应预设样式、在飞书中层级过深，**一律用加粗段落替代**：

```markdown
**字段说明：**  （代替 #### 字段说明）
```

---

## 2. 表格规范

**允许：**
- 简单行列表格，建议最多 5 列
- 单元格内：纯文本、加粗文字、行内代码

**禁止：**
- 单元格内嵌套列表
- 使用 `<br>` 在单元格内换行（飞书/Word 均不支持）
- 合并单元格（标准 Markdown 不支持）
- 单元格内放置代码块

表头分隔符统一使用 `|---|---|` 格式：

```markdown
| 字段 | 内容 |
|---|---|
| need_id | REQ-20240315-001 |
```

---

## 3. 代码块使用规范

**代码块（` ``` `）仅用于：**
- 编程代码（Java、JavaScript、SQL 等）
- Shell 命令
- JSON / YAML 配置
- 接口请求 / 响应示例

**禁止将以下内容放入代码块：**
- 评估规则或判定逻辑的说明文字
- 普通段落内容
- 非技术性列表

**不合规（说明文字误放代码块）：**

````
```
业务分 ≤ 7 且 技术分 ≤ 6，无3分项 → S
业务分 8-10 或 技术分 7-8，最多1个3分项 → M
```
````

**修正：改为表格或普通列表。**

```markdown
| 条件 | 等级 |
|---|---|
| 业务分 ≤ 7 且 技术分 ≤ 6，无3分项 | **S** |
| 业务分 8-10 或 技术分 7-8，最多1个3分项 | **M** |
```

---

## 4. 占位符规范

模板中的填写指引分两类，AI 在生成**最终输出文档**时必须正确处理：

### A. AI 负责替换的占位符（模板内用 `<!-- -->` 标注）

这类占位符在生成最终文档时**必须全部替换为实际内容或删除**，不得保留在输出文件中。

```markdown
<!-- 说明当前存在什么问题。回答"为什么要做这件事" -->
→ 替换为：当前用户查询接口 P99 延迟达 3 秒，影响核心下单流程。
```

### B. AI 无法推断、待人工填写的字段

使用可见占位符，标明原因：

```markdown
**提需人**：（待填写）
**期望上线日期**：（待确认 — 提需方尚未给出具体日期）
```

### 最终输出文档要求

生成 `docs/requirements/backlog/REQ-XXXXXXXX.md` 等文件时：

1. 所有 `<!-- ... -->` 注释 → 替换为实际内容，或整行删除
2. 无法推断的字段 → 使用 `（待填写）` 标注
3. 模板中的"填写说明"类纯指引文字（如"填写说明：根据实际需求..."）→ 删除

---

## 5. 引用块规范

引用块 `>` 用于：
- 注意事项或警告
- 操作指引
- 来源说明

```markdown
> **注意**：以下问题需提需方确认后才能进入技术设计阶段。
```

禁止将大段正文放入引用块。

---

## 6. 列表规范

- 有序列表：`1. 2. 3.`（渲染器自动处理序号，无需手动维护）
- 无序列表：`- ` 前缀
- 任务列表：`- [ ]` / `- [x]`（飞书和 Word 均支持）
- 嵌套不超过 2 层

---

## 7. 禁止使用的元素

| 禁止元素 | 原因 | 替代方案 |
|---|---|---|
| `<br>` `<div>` 等 HTML 标签 | 飞书不渲染、Word 可能乱码 | 使用空行或 `---` 分隔 |
| `<!-- 注释 -->` 出现在最终输出中 | 部分工具将注释渲染为可见文字 | 替换为实际内容或 `（待填写）` |
| H4 及以下标题 `####` | Word 无对应预设样式 | 改用加粗段落 |
| 代码块包裹非代码内容 | 转换后变为等宽字体，格式怪异 | 改为普通文本、表格或引用块 |
| 单元格内 `<br>` 换行 | 飞书/Word 不支持 | 拆分为多行 |
| 超宽代码行（>100字符/行） | Word / 飞书横向溢出 | 折行处理 |

---

## 8. 导出前检查清单

将 Markdown 文档导出为 Word 或导入飞书文档之前，逐项确认：

- [ ] 文档只有一个 H1 标题
- [ ] 所有标题层级不超过 H3
- [ ] 文档中无 `<!-- ... -->` 注释残留
- [ ] 代码块仅用于代码、命令、配置，非代码内容已改为普通文本或表格
- [ ] 文档中无任何 HTML 标签（`<br>`、`<div>` 等）
- [ ] 表格每行列数一致，无合并单元格
- [ ] 列表嵌套不超过 2 层
- [ ] 模板填写指引文字已全部删除
