# AI编程规范模板

## 项目介绍

这是一个专为AI编程设计的规范驱动开发工具模板，结合OpenSpec和Superpowers方法论，通过轻量级的配置和标准化目录结构，让规范驱动流程可快速落地。

## 核心优势

- **支持主流AI工具**：兼容Claude Code、Cursor等主流AI工具
- **无需复杂配置**：新手开发者也能快速上手
- **完整开发闭环**：涵盖"环境准备-流程执行-归档沉淀"三大阶段
- **可复用技能**：支持动态加载和组合技能

## 如何让 Claude Code 识别此模板

1. **确保目录结构正确**：`.ai-config` 目录必须位于项目根目录
2. **使用标准文件格式**：所有配置文件使用标准的 Markdown 格式
3. **明确的指令格式**：在与 Claude Code 交互时，明确指定使用此模板

### 推荐的 Claude Code 指令

```
# 让 Claude Code 按照此模板的规范进行开发
请参考 .ai-config 目录下的规范文件，按照以下步骤进行开发：

1. 首先生成开发计划
2. 然后生成前端原型
3. 接着实现后端API
4. 最后进行代码审查和测试

技术栈：[您的技术栈选择]
```

## 快速开始

1. **克隆此模板**到您的项目根目录
2. **配置环境**：根据项目需求修改 `.ai-config` 目录下的配置文件
3. **开始开发**：按照 `EXAMPLE.md` 中的示例使用技能进行开发

## 核心功能

- **规范驱动**：基于 `rules/` 目录下的规范文件，确保代码质量和一致性
- **技能复用**：使用 `skills/` 目录下的技能，提高开发效率
- **代理协作**：利用 `agents/` 目录下的代理配置，实现专业分工
- **自动化流程**：通过 `hooks.json` 配置自动化触发行为

## 扩展能力

1. **添加新技能**：在 `skills/` 目录下创建新的技能文件夹
2. **配置新代理**：在 `agents/` 目录下创建新的代理配置文件
3. **自定义规则**：修改 `rules/` 目录下的规范文件
4. **集成外部工具**：在 `mcp/` 目录下配置外部工具

## 最佳实践

- **定期更新技能库**：保持技能的有效性和先进性
- **严格遵循规范**：确保代码质量和一致性
- **利用自动化**：通过钩子配置提高开发效率
- **持续改进**：根据项目需求不断优化和扩展模板

## 适用场景

- **新项目初始化**：为新项目建立规范和流程
- **现有项目改造**：为现有项目引入规范和工具
- **团队协作**：统一团队的开发规范和工作流程
- **AI辅助开发**：与Claude Code、Cursor等AI工具配合使用

## 支持的AI工具

- **Claude Code**：通过扫描 `.ai-config` 目录识别模板
- **Cursor**：通过配置文件识别和使用模板
- **Trae IDE**：完全兼容，支持技能和代理的使用
- **其他AI工具**：支持标准的Markdown格式配置
