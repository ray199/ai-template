# 工作流快速参考卡片

## 🚀 30秒快速开始

```bash
/intake
[粘贴你的需求]
```

系统会自动：
1. 识别项目上下文
2. 识别需求质量 (T1-T4)
3. 生成结构化文档
4. 评估工作量等级 (S/M/L/XL)
5. 提示需要补充的信息

## 📊 工作量等级速查

| 等级 | 判断标准 | 工期 | 输出物数 | 人工审查 |
|---|---|---|---|---|
| **S** | 简单<br/>1-2页面<br/>加字段 | 1-2天 | 3-4 | 无 |
| **M** | 标准<br/>3-4页面<br/>新表1-2张 | 3-5天 | 5-6 | 1人 |
| **L** | 复杂<br/>5+页面<br/>跨模块<br/>有风险 | 1-2周 | 7-8 | 1人(终审) |
| **XL** | 超大<br/>整体重构<br/>多模块 | 4周+ | 10+ | 3人(团队) |

## 🎯 按需求类型快速判断

| 需求类型 | 工作量 | 示例 |
|---|---|---|
| 新增字段 | S | 给User表加名字字段 |
| 新增页面 (2-3个) | M | 新增权限管理后台 |
| 新增页面 (5+个) + 新表 | L | 订单系统改造 |
| 系统级重构 | XL | 电商平台从零到一 |

## 📋 各阶段关键指令

### 阶段1: 需求接入 (自动5步)
```bash
/intake              # ← 在这里粘贴需求内容
/intake-list         # 查看所有需求
/intake-check REQ-X  # 检查完整性
/intake-update REQ-X # 补充缺失字段
```

### 阶段2-3: 设计+编码 (M/L/XL需要)
```bash
/design REQ-X        # 生成设计文档
/code REQ-X          # 生成源代码
```

### 阶段4: 测试+审查
```bash
/test REQ-X          # 执行测试
/review REQ-X        # 代码审查
/review-fix REQ-X    # 提交修复后重新审查
```

### 阶段5: 交付上线
```bash
/deliver REQ-X       # 生成交付文档
```

## 🔍 常见问题速解

**Q: 我的需求是一句话，够吗？**  
A: 完全够！系统会提3个引导问题，回答即可自动生成完整文档。

**Q: 怎么判断我的需求是S还是M？**  
A: 看涉及的页面数和数据库表数。<3页面新字段 = S；3-4页面新表 = M。

**Q: S等级真的不需要设计吗？**  
A: 是的。S等级很简单，设计文档反而浪费时间。直接编码最高效。

**Q: 代码审查要多久？**  
A: S无需人工审查(自动检查)；M需要1人初检，L需要终审，XL需要团队评审。

**Q: 我能跳过测试吗？**  
A: 不能。所有阶段都要测试。单元测试覆盖率要≥80%。

**Q: 回滚脚本怎么写？**  
A: 根据你的迁移脚本反向操作。数据库升级了就写降级脚本，依赖引入了就写移除脚本。

## 💡 3个核心建议

1. **T1就启动** - 不要等需求完美，一句话就能开始。系统会引导补充。

2. **信任自动补全** - 技术栈、影响模块等会自动推断。检查一下就行，无需重新填写。

3. **等级很关键** - S/M/L/XL决定整个输出规格。评估准确能省50%的时间。
